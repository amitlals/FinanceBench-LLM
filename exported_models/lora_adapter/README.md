# FinanceBench-LLM: LoRA Adapter for Financial QA

A LoRA adapter fine-tuned on the [PatronusAI/financebench](https://huggingface.co/datasets/PatronusAI/financebench) dataset for financial question answering over SEC filings (10-K, 10-Q).

Built following the **NVIDIA DLI "Evaluation and Light Customization of Large Language Models"** course using NVIDIA NIM and NeMo Customizer.

## Results

| Model Configuration | Exact Match | F1 Score | Faithfulness | Correctness | ELO Rating |
|---------------------|-------------|----------|--------------|-------------|------------|
| Base (Llama-3.1-8B) | 0.23 | 0.41 | 3.2/5 | 2.8/5 | 835 |
| + ICL (5-shot) | 0.34 | 0.56 | 3.9/5 | 3.6/5 | 1023 |
| **+ LoRA Fine-tuned** | **0.52** | **0.71** | **4.4/5** | **4.2/5** | **1142** |

**Key improvements**: +126% Exact Match, +73% F1, +50% Correctness vs. base model.

## Training Details

- **Base model**: meta-llama/Llama-3.1-8B-Instruct
- **Method**: LoRA (Low-Rank Adaptation)
- **LoRA rank (r)**: 16
- **LoRA alpha**: 32
- **LoRA dropout**: 0.05
- **Target modules**: q_proj, v_proj, k_proj, o_proj
- **Training epochs**: 2
- **Batch size**: 4
- **Learning rate**: 2e-4
- **Max sequence length**: 1024
- **Dataset**: PatronusAI/financebench (120 training examples, 30 test)
- **Training infrastructure**: NVIDIA NeMo Customizer

## Usage

```python
from transformers import AutoModelForCausalLM, AutoTokenizer
from peft import PeftModel

# Load base model
base_model = AutoModelForCausalLM.from_pretrained(
    "meta-llama/Llama-3.1-8B-Instruct",
    torch_dtype="auto",
    device_map="auto",
)

# Load LoRA adapter
model = PeftModel.from_pretrained(
    base_model, "amitlal/financebench-lora-adapter"
)

# Load tokenizer
tokenizer = AutoTokenizer.from_pretrained(
    "meta-llama/Llama-3.1-8B-Instruct"
)

# Query
prompt = "What was Apple's total revenue for fiscal year 2023?"
inputs = tokenizer(prompt, return_tensors="pt").to(model.device)
outputs = model.generate(**inputs, max_new_tokens=256)
print(tokenizer.decode(outputs[0], skip_special_tokens=True))
```

## Evaluation Methodology

- **Automated metrics**: Exact Match (normalized), token-level F1
- **LLM-as-a-Judge**: Llama-3.1-70B on correctness, faithfulness,
  conciseness (1-5 scale)
- **ELO ranking**: 1000 pairwise comparisons, K-factor=32
- **Experiment tracking**: MLflow with full artifact logging

## Limitations

- Trained on 120 examples — may not generalize to all financial domains
- Optimized for SEC filing QA (10-K, 10-Q) — not investment advice
- English only
- Requires meta-llama/Llama-3.1-8B-Instruct base model (gated)
